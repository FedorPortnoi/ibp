Модель: `claude-haiku-4-5-20251001`  
Цены: **$0.80 / 1M input tokens, $4.00 / 1M output tokens**

Из `claude_integration.py` — ровно 4 вызова за проверку:

| Вызов | Когда | max_tokens | Примерно input | Примерно output | Стоимость |

| `summarize_court_cases` | Если есть судебные дела (~30% проверок) | 1024 | ~700 | ~300 | ~$0.00176 |
| `generate_behavioral_summary` | Если VK стена читаема (~40% проверок) | 384 | ~500 | ~250 | ~$0.0014 |
| `generate_risk_narrative` | **Всегда** | 256 | ~500 | ~150 | ~$0.0010 |
| `generate_executive_summary` | **Всегда** | 384 | ~800 | ~300 | ~$0.00184 |

**Итого на проверку:**
- Всегда: $0.0010 + $0.00184 = **$0.00284**
- Условные (weighted): courts ×0.30 = $0.000528; behavioral ×0.40 = $0.00056
- **Среднее: ~$0.004/проверку** (диапазон $0.003–$0.006)

> Предыдущая оценка $0.009 — завышена вдвое.

## Узкие места по масштабу

### 100 пользователей (500 чек/мес)
- **Claude $2/мес — минимально.** Основные затраты — сервер $6.
- parser-api.com: 200 бесплатных req/мес → хватает на 100 проверок (500 чек × 2 = 1000 calls → нужен платный). Но ключ уже оплачен — просто скопировать в ibp/.env.
- Bottleneck: ничего критичного. Можно запустить.

### 1 000 пользователей (5 000 чек/мес)
- **Bottleneck:** Playwright (Search4Faces без API-ключа) — 1 браузер = 2-3 мин/проверку
- Нужен Search4Faces API (~$40-80/мес) или 3+ Playwright worker'а → нужен CX32+
- parser-api.com: 5 000 чек × 2 = 10 000 calls/мес → нужен платный тариф (~$30/мес)

### 2 000+ пользователей (10 000+ чек/мес)
- Playwright не масштабируется → **обязателен Search4Faces API** ($40-80/мес)
- Нужна очередь (Celery + Redis или RQ)
- SQLite → PostgreSQL обязательно

### 5 000+ пользователей (25 000+ чек/мес)
- Горизонтальный scaling: 2+ worker-сервера
- parser-api.com может не иметь тарифов для 50 000 req/мес → альтернативы (собственный прокси)
