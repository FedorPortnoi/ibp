  <name>Task 1: Create Yandex reverse image search service</name>
  <files>app/services/yandex_image_search.py</files>
  <action>
Create `app/services/yandex_image_search.py`:

1. Imports: `requests`, `re`, `os` from stdlib; `BeautifulSoup` from bs4

2. Constants:
   ```python
   YANDEX_UPLOAD_URL = 'https://yandex.com/images/search'

   HEADERS = {
       'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36',
       'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8',
       'Accept-Language': 'en-US,en;q=0.9,ru;q=0.8',
   }

   # Social media domains to extract from results
   SOCIAL_DOMAINS = ['vk.com', 'ok.ru', 'instagram.com', 'facebook.com', 't.me', 'twitter.com', 'x.com']
   ```

3. Helper function `_detect_platform(url: str) -> str`:
   - Map domains to platform names: vk.com->VK, ok.ru->OK, instagram.com->Instagram, etc.
   - Return 'Web' for unknown domains

4. Main function `yandex_reverse_image_search(image_path: str) -> list`:
   - Validate image_path exists, return empty list if not
   - Open image file in binary mode
   - POST to YANDEX_UPLOAD_URL with:
     - files={'upfile': ('image.jpg', file_handle, 'image/jpeg')}
     - data={'rpt': 'imageview'}
     - headers=HEADERS
     - timeout=30
     - allow_redirects=True
   - Check for CAPTCHA in response (if 'captcha' or 'SmartCaptcha' in response.text, log warning and return [])
   - Parse response with BeautifulSoup
   - Extract all links from response
   - For each link href:
     - If href contains any SOCIAL_DOMAIN, add to results with:
       - platform: from _detect_platform()
       - url: the href
       - source: 'yandex_image'
       - title: link text or parent element text (truncate to 100 chars)
   - Return list of result dicts

Key pitfall to avoid: Yandex may return CAPTCHA page. Detect this by checking for 'captcha' in response and return empty list gracefully instead of crashing.

Note: Yandex reverse image search flow can be complex (redirects, JS). Start with simple POST and iterate if needed. The goal is social profile URLs from the response, not perfect parsing.
