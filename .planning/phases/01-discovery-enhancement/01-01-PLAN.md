<task type="auto">
  <name>Task 1: Create VK username search service</name>
  <files>app/services/vk_search.py</files>
  <action>
Create `app/services/vk_search.py` following the exact pattern of `telegram_search.py`:

1. Imports: `requests`, `re`, `time`

2. HEADERS constant with realistic browser User-Agent:
   - User-Agent: Chrome 120 on Windows
   - Accept: text/html
   - Accept-Language: en-US,en;q=0.5,ru;q=0.3

3. `check_vk_username(username: str) -> dict` function:
   - GET request to `https://vk.com/{username}` with 10s timeout
   - Check response content (NOT just status code) for error indicators:
     - "page not found", "user not found", "deleted", "deactivated", "banned", "the page you requested"
   - If error indicator found, return `{'exists': False}`
   - If valid profile, extract:
     - display_name from `<title>` tag (strip " | VK" suffix)
     - photo_url from page_avatar img src (optional, may be None)
   - Return dict with: platform='VK', username, url, display_name, photo_url, exists=True, source='vk_direct'

4. `check_vk_usernames(usernames: list) -> list` function:
   - Iterate through usernames
   - Call `check_vk_username()` for each
   - Add 0.5 second delay between requests (rate limiting)
   - Return list of results where exists=True

Key pitfall to avoid: VK returns 200 OK even for non-existent profiles. Must check HTML content for error messages, not HTTP status code.
